Control Mechanisms¶

Overview¶

ControlMechanisms monitor the outputState(s) of one or more ProcessingMechanisms in a System and use this information to regulate the value of Control Signal projections to other ProcessingMechanisms in that system. .

Creating A ControlMechanism¶

ControlMechanisms can be created by using the standard Python method of calling the subclass for the desired type. A ControlMechanism is also created automatically whenever a system is created (see Creating a System), and assigned as the controller for that system (see _System_Execution_Control).

MonitoredOutputStates¶

The outputStates monitored by a ControlMechanism can be specified in its monitored_output_states argument when it is created, or in it its params dictionary using the MONITORED_OUTPUT_STATES. The parameter must be a list, each item of which is one of the following:

  • a string that is the name of an outputState
  • an outputState or mechanism object

  • a tuple with three items, in the order listed below:

    • the name of an outputState, an outputState or mechanism object, or a specification dictionary for one;
    • an exponent (int) - exponentiates the value of the outputState;
    • a weight (int) - multiplies the value of the outState;
  • a MonitoredOutputStatesOption (enum) value, which specifies monitoring all of the

TERMINAL mechanisms of the system, using either the PRIMARY_OUTPUT_STATES or ALL_OUTPUT_STATES for each.

Execution¶

The ControlMechanism of a system is always the last to be executed (see System Control). A ControlMechanism’s function takes as its input the values of the outputStates specified in its MONITORED_OUTPUT_STATES parameter, and uses those to determine the value of its Control Signal projections. In the next round of execution, each ControlSignal’s value is used by the Parameter State to which it projects, to update the corresponding parameter of the recieving mechanism’s function.

Note

A Parameter State that receives a Control Signal projection does not update its value until its owner mechanism executes (see Lazy_Evaluation for an explanation of “lazy” updating). This means that even if a ControlMechanism has executed, a parameter that it controls will not assume its new value until the corresponding receiver mechanism has executed.
