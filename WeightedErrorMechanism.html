WeightedErrorMechanism¶

Overview¶

A WeightedErrorMechanism monitors the outputState of it’s errorSource, a Processing Mechanisms that projects to another mechanism in a process or system. It computes a weighted_error_signal, which is the contribution of each element of the output of the errorSource to the error for the output of the mechanism to which the errorSource projects. The weighted_error_signal returned by the function of the WeightedErrorMechanism is used by a LearningProjection to adjust a MappingProjection to the errorSource, so as to reduce its future contribution to the error generated by the network.

Creating A WeightedErrorMechanism¶

A WeightedErrorMechanism can be created directly by calling its constructor, or using the mechanism function and specifying WeightedErrorMechanism as its mech_spec argument. It can also be created by in context specification of a LearningProjection for a MappingProjection; that MappingProjection must project to a ProcessingMechanism that is not a TERMINAL mechanism. One or more WeightedErrorMechanisms, are also created automatically, along with associated LearningProjections, when learning is specified for a process or system with any ProcessingMechanisms that are not TERMINAL or SINGLETON mechanisms.

Structure¶

A WeightedErrorMechanism has a single inputState and a single outputState (labelled WEIGHTED_ERROR). It also has four primary attributes: an

  • errorSource: the ProcessingMechanism for which the WeightedErrorMechanism computes the weighted_error_Signal;
  • next_mechanism: a ProcessingMechanism to which the errorSource projects;
  • projection_to_next_mechanism: the MappingProjection from the errorSource to the next_mechanism;
  • error_signal: the error for the value of the next_mechanism;
  • weighted_error_Signal: the error for the value of the errorSource, weighted by the contribution that each element of its elements makes to the error_signal.

The WeightedErrorMechanism receives a MappingProjection to its inputState from the errorSource, and its WEIGHTED_ERROR outputState is assigned a LearningProjection to the MappingProjection it is being used to train (and that projects to the errorSource)

Execution¶

A WeightedErrorMechanism always executes after the mechanism it is monitoring. It’s function computes its weighted_error_signal: the contribution that each element of the errorSource makes to the error_signal (i.e., the error for the value of the next_mechanism), weighted both by the strength of its association to each element of the next_mechanism (specified in projection_to_next_mechanism), and the differential of the function of next_mechanism. This implements a core computation of the Generalized Delta Rule implemented by the BackPropagation learning function. The function of the WeightedErrorMechanism returns the weighted_error_signal, that is assigned to its value and outputValue attributes, and the value of of its WEIGHTED_ERROR outputState.
